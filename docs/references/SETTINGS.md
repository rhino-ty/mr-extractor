## 패키지 버전 현황

```
┌─────────────────────────────────────────────────────────┐
│  패키지 버전                              [🔄 새로고침]  │
├──────────────┬──────────┬──────────┬────────────────────┤
│  패키지       │ 설치됨   │ 최신     │ 상태               │
├──────────────┼──────────┼──────────┼────────────────────┤
│  demucs      │ 4.0.1    │ 4.0.1    │ ✅ 최신            │
│  yt-dlp      │ 2024.11  │ 2025.03  │ 🔄 업데이트 가능   │
│  ...         │          │          │                    │
└─────────────────────────────────────────────────────────┘
```

상태:

- ✅ 최신 — 범위 내 최신
- 🔄 업데이트 가능 — 범위 내 더 높은 버전 있음
- 🔒 메이저 업데이트 있음 — constraint 밖 (자동 업데이트 안 함)
- ❌ 미설치

[선택 업데이트]: 🔄 상태만 체크박스 활성, 🔒는 비활성.

## 저장 정책

```python
STORAGE_POLICY = {
    "keep_all":      "스템 전체 저장",        # 기본값
    "session_only":  "임시 저장",             # 재생 후 스템 삭제
    "inst_only":     "반주 파일만 만들기",     # 스템 없음
}
```

- keep_all: 곡당 약 200~600 MB, 믹서 항상 사용 가능
- session_only: 반주 파일은 유지, 나중에 믹서 쓰려면 재처리
- inst_only: 곡당 약 10~30 MB, 믹서 사용 불가

## 저장 공간

```
┌──────────────────────────────────────────────────────────┐
│  저장 공간                                                │
├──────────────────────────────────────────────────────────┤
│  출력 폴더     ~/Desktop/MR Extractor/   [📁 폴더 열기]  │
│  전체 사용량   2.3 GB  (파일 14개)                        │
│                                                          │
│    스템 파일   1.8 GB  ███████████████████████████  78%  │
│    반주 파일   0.4 GB  █████  17%                        │
│    기타        0.1 GB  █  5%                             │
│                                                          │
│  [🗑 전체 삭제]      [🧹 고아 파일 정리]                  │
└──────────────────────────────────────────────────────────┘
```

- **전체 삭제**: 확인 다이얼로그 + "히스토리도 함께 삭제" 체크박스
- **고아 파일 정리**: 히스토리에 없는 파일 탐지 → 선택 삭제
- 용량 계산: `StorageScanWorker(QThread)` 백그라운드

### 모델 캐시

demucs 모델은 `~/.cache/torch/hub/checkpoints/`에 자동 다운로드됨.
htdemucs_ft는 Bag of 4 모델이라 약 300~400MB 차지.
저장 공간 계산 시 이 경로도 포함 가능 (선택적).
